## ⚡ 3단계로 배포하기 (총 5분)

### 1️⃣ GitHub 업로드 (2분)

```bash
cd photo-studio
git init
git add .
git commit -m "Initial commit"

# GitHub에서 새 리포지토리 생성 후:
git remote add origin https://github.com/당신의사용자명/photo-studio.git
git branch -M main
git push -u origin main
```

### 2️⃣ Vercel 연결 (1분)

1. https://vercel.com 접속
2. "Sign Up with GitHub" 클릭
3. "Import Project" 클릭
4. `photo-studio` 리포지토리 선택

### 3️⃣ 환경 변수 설정 (2분)

**Environment Variables** 섹션에 추가:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGc...
GMAIL_USER=your-email@gmail.com
GMAIL_APP_PASSWORD=xxxx xxxx xxxx xxxx
```

**"Deploy" 버튼 클릭!** → 2-5분 후 배포 완료! 🎉

## 🔑 환경 변수 준비하기

배포 전에 이 값들을 미리 준비하세요:

### 1. Supabase 키

**찾는 방법:**
1. https://supabase.com/dashboard 로그인
2. 프로젝트 선택
3. **Settings** → **API**
4. 복사:
   - **Project URL** → `NEXT_PUBLIC_SUPABASE_URL`
   - **anon public** 키 → `NEXT_PUBLIC_SUPABASE_ANON_KEY`

### 2. Gmail 앱 비밀번호

**찾는 방법:**
1. Gmail 계정 설정
2. 보안 → 2단계 인증 활성화
3. 앱 비밀번호 생성 (16자리)
4. 복사:
   - 이메일 주소 → `GMAIL_USER`
   - 앱 비밀번호 → `GMAIL_APP_PASSWORD`

**자세한 방법:** [GMAIL_SETUP.md](./GMAIL_SETUP.md)

### 3. 관리자 설정 (선택사항)

```bash
NEXT_PUBLIC_ADMIN_EMAILS=admin@yourdomain.com
NEXT_PUBLIC_ADMIN_PATH=my_secret_admin_2024
```

## 🐛 문제 해결

### 빌드 실패?
```bash
# 로컬에서 먼저 테스트
npm run build
```

### 사이트가 안 열려요?
- Vercel 대시보드에서 빌드 로그 확인
- 환경 변수가 모두 입력되었는지 확인

### 이미지가 안 나와요?
- Supabase Storage 버킷이 **public**인지 확인
- RLS 정책 확인

### 이메일이 안 와요?
- Gmail 앱 비밀번호 확인 (16자리, 공백 포함)
- 스팸 메일함 확인
- Vercel 함수 로그 확인

**더 많은 해결책:** [DEPLOYMENT_GUIDE.md - 문제 해결](./DEPLOYMENT_GUIDE.md#-문제-해결)
